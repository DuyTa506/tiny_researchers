## Context Summary

Đang xây dựng **Claw Researcher** — một AI agent system cho nhà nghiên cứu với 2 phase:
- **Phase 1 (Research Intelligence):** Brainstorm, Survey, Paper Synthesis, Gap Finding, Report, Future Directions
- **Phase 2 (Experiment Reproduction):** Parse paper → Setup env → Find dataset → Generate code → Run → Verify

Project dùng **nanobot-style architecture** (đã có sẵn trong `nanobot/` subfolder) làm reference, với agent loop riêng tại `claw/`.

Tech stack: Python 3.11, uv, litellm (multi-provider LLM), anthropic SDK, httpx, loguru, typer, streamlit, pytest+asyncio.

## Key Decisions Made

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| Nanobot-style architecture (skills as SKILL.md files) | Reuse existing pattern trong project, dễ extend | LangChain/CrewAI agents |
| Tools không có constructor args | Simpler instantiation, config qua env vars | Pass workspace/config vào constructor |
| Code gen tạo scaffold + TODOs thay vì full implementation | LLM hallucination risk cao cho novel architectures; human reviews TODOs | Full auto-gen via LLM |
| dataset_download generates scripts thay vì download trực tiếp | Agent có thể inspect script trước khi chạy; tránh timeout trong tool execution | Download trực tiếp trong tool |
| ar5iv.org là primary source cho paper_fetch | HTML text quality tốt hơn PDF parsing, không cần Nougat/Grobid binary | PyMuPDF, Nougat, Grobid |
| litellm cho LLM abstraction | Multi-provider support (Claude, GPT, DeepSeek) với single interface | Anthropic SDK trực tiếp |
| File-based memory (MEMORY.md + HISTORY.md) | Zero external dependencies, human-readable, grep-able | SQLite, Redis, vector DB |
| pytest-asyncio với `asyncio_mode = "auto"` | Tất cả async tools test được trực tiếp không cần `asyncio.run()` | Manual event loop |
